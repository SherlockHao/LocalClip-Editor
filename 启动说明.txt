======================================
  LocalClip Editor - 启动说明
======================================

【推荐方式】静默启动（无黑色窗口）
------------------------------------
双击运行： start.vbs

✓ 无黑色命令行窗口
✓ 自动等待服务就绪
✓ 自动打开浏览器
✓ 显示启动状态提示


【标准方式】命令行启动
------------------------------------
双击运行： start.bat

✓ 显示详细启动日志
✓ 自动检测服务状态
✓ 自动打开浏览器
✓ 可查看实时运行信息


======================================
  启动时间说明
======================================

首次启动：约 30-60 秒
- 需要安装前端依赖（npm install）
- 需要启动后端服务
- 需要启动前端开发服务器

后续启动：约 10-15 秒
- 依赖已安装，直接启动服务


======================================
  访问地址
======================================

前端界面： http://localhost:5173
后端API：  http://localhost:8000/docs


======================================
  如何停止服务
======================================

方法1：关闭命令行窗口
- 找到 "LocalClip-Backend" 窗口并关闭
- 找到 "LocalClip-Frontend" 窗口并关闭

方法2：使用任务管理器
- 结束 python.exe 进程（后端）
- 结束 node.exe 进程（前端）


======================================
  常见问题
======================================

Q: 双击 start.vbs 后无反应？
A: 1. 检查是否已安装 Node.js
   2. 检查是否已配置 Conda 环境 'ui'
   3. 以管理员身份运行

Q: 浏览器打开后显示"无法连接"？
A: 服务可能仍在启动中，请等待 1-2 分钟后刷新页面

Q: 提示端口被占用？
A: start.bat 会自动清理端口，如果仍有问题，请手动关闭占用进程：
   - 端口 8000：后端服务
   - 端口 5173：前端服务

Q: 首次启动时间很长？
A: 正常现象，首次需要安装依赖（npm install），请耐心等待


======================================
  系统要求
======================================

必需：
- Windows 10 或更高版本
- Node.js 16 或更高版本
- Anaconda/Miniconda（包含 'ui' 环境）
- Python 3.10+

推荐：
- 8GB 以上内存
- SSD 硬盘
- 稳定的网络连接（用于首次安装依赖）


======================================
  技术支持
======================================

遇到问题请检查：
1. backend 窗口的错误日志
2. frontend 窗口的错误日志
3. 网络连接是否正常
4. 依赖是否完整安装
